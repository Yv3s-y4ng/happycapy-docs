## What are Automations?

Automations is a set of automated rule systems consisting of **triggers** and **actions**:

<Columns cols={2}>
  <div>
    **Trigger**

    Under what circumstances automation starts:
    - Receiving emails
    - File uploads
    - Scheduled tasks
    - Webhook events
  </div>

  <div>
    **Action**

    What operations are automatically performed after triggering:
    - Generating reports
    - Sending notifications
    - Processing data
    - Updating databases
  </div>
</Columns>

## Detailed Example: Daily News Selection

<Note>
**Setup time**: 2 minutes | **Follow-up**: Runs automatically every day

**Pain point**: I open 10 websites every day to read news. There is too much information to read, and valuable content is drowned.
</Note>

### Automation Configuration

```
Help me set up an automation task:

Name: Daily News Digest

Trigger: Every day at 8:00 AM

Execution flow:
1. Fetch trending news in these areas:
   - AI and technology
   - Startups and business
   - Product and design

2. Smart filtering:
   - Exclude entertainment gossip
   - Exclude clickbait and ads
   - Keep only in-depth content

3. Summarize each news item with AI (50-word key takeaway)

4. Sort by importance (major industry events first)

5. Generate a well-designed HTML daily brief

6. Send it to my email

Help me quickly stay updated on industry trends during my commute every day!
```

<Tip>
After the setup is completed, happycapy will automatically execute at 8 AM every day. You only need to open the email on your phone and you can read the important information of the day in 5 minutes!
</Tip>

## Trigger Types

Automations supports various trigger types:

<CardGroup cols={2}>
  <Card title="Schedule" icon="calendar">
    Run at specific times or intervals (daily, weekly, monthly)
  </Card>

  <Card title="File Events" icon="file">
    Triggered when files are uploaded or modified
  </Card>

  <Card title="Email" icon="envelope">
    Triggered when emails are received (specific sender, subject, etc.)
  </Card>

  <Card title="Webhooks" icon="webhook">
    Triggered by external services via HTTP requests
  </Card>

  <Card title="Manual" icon="hand-pointer">
    Run on demand with a button click
  </Card>

  <Card title="Chain" icon="link">
    Triggered when another automation completes
  </Card>
</CardGroup>

## Tips for Success

<Tip>
**Natural language works**: You don't need to know technical details. Just describe what you want to happen in plain language, and happycapy will configure the automation for you.
</Tip>

<Tip>
**Iterate and improve**: Start with a basic automation and refine it over time based on results. You can always edit and enhance existing automations.
</Tip>

<Warning>
**Be mindful of frequency**: Automations that run too frequently (e.g., every minute) can consume significant resources. Choose appropriate intervals for scheduled automations.
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion icon="circle-question" title="Automation didn't trigger">
    - Check that the automation is activated (toggle switch is on)
    - Verify the trigger conditions are correct
    - Review automation logs for errors
    - Check that required permissions are granted
  </Accordion>

  <Accordion icon="circle-xmark" title="Action failed to complete">
    - Check automation logs for error messages
    - Verify API keys and credentials are valid
    - Ensure required files or data are available
    - Test the action manually to isolate the issue
  </Accordion>

  <Accordion icon="clock-rotate-left" title="Automation runs too slowly">
    - Simplify complex processing steps
    - Consider splitting into multiple automations
    - Reduce the amount of data being processed
    - Use more efficient file formats
  </Accordion>
</AccordionGroup>
