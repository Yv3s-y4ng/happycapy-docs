## HappyCapy 是什么?

一台**运行在浏览器里的 Agent 原生计算机**。

由 Claude Code 驱动,为所有人设计。

不需要配环境,不需要学命令行,不需要懂技术。打开浏览器,说出你的需求,看着它帮你完成工作。

生成图片、剪辑视频、处理数据、做网页、写论文...以前需要学十几个软件才能做的事,现在只需要说出来。

过去一年,我们看到 AI Agent 能做的事越来越多。但大部分 Agent 工具都是为开发者设计的——要会用命令行,要懂得配置环境,要知道怎么处理安全问题。我们觉得,Agent 不应该只属于懂技术的人。所以我们做了 HappyCapy。

### 传统电脑的逻辑
装软件 → 学软件 → 用软件完成任务

### Agent 原生计算机的逻辑
说需求 → AI 调用工具 → 直接给结果

## HappyCapy 背后的故事

### 我们自己先变成了 AI native 团队

2025 年 1 月,Claude Code 出来了。它改变了编码的模式——开发者可以用自然语言写代码,效率提升了好几倍。

我们团队用得很爽。但同时我们也在想:**为什么 AI 的门槛还是这么高?**

想用好 AI,要么懂命令行、会配环境、学提示词,要么就只能停留在聊天窗口里,很难发挥出 AI 的全部能力。为什么不能像用电脑一样简单?

**AI 应该是平权的。**

不管你是什么职业、什么背景——每个人都应该平等地享受 AI 带来的生产力提升。

所以我们给自己做了一个工具。用上之后,整个团队的工作方式改变了。不懂技术的人也能自己写代码、做网页、处理数据。每个人都能把想法变成现实。

**每个人都成了超级个体。**

然后我们把这套工作方式产品化了,就是 HappyCapy。

### 为什么是 Capy(卡皮巴拉 - Capybara - 水豚)

水豚温和、友善、与所有动物和谐相处。你能在网上看到水豚和猴子、鸟、鳄鱼、甚至猫狗都能玩在一起。

这正是我们想要的:一个和所有人都能"相处"的 AI 工具。

**Happy + Capy = 快乐的水豚 = AI 应该使人变得快乐而不是焦虑。**

我们希望大家用 HappyCapy 的时候,就像养一只卡皮巴拉一样——chill,没有焦虑,回归到生活本身。

## 我们的愿景

**让所有人都用上 AI,自动化自己的工作流,减少人的重复性工作。**

**HappyCapy, HappyYou.**

我们不觉得 AI 是用来取代人的。我们觉得 AI 是用来让人更像人的——把重复的、烦人的、消耗时间的事情交给 AI,把创意、判断、决策留给自己。

你只需要专注于**做什么**,而不是**怎么做**。

不用纠结今天该用哪个模型,不用被每天 AI 新突破的新闻搞得焦虑。

不管你是创作者、开发者、学生、上班族、还是自由职业者——只要你有想法,HappyCapy 就能帮你把想法变成现实。
