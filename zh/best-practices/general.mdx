## 高效使用的黄金法则

### 1. 清晰描述你的目标,而非过程

<Columns cols={2}>
  <div>
    **不好的提示:**
    ```
    先用 pandas 读取 CSV,
    然后用 matplotlib 画图
    ```
  </div>
  <div>
    **好的提示:**
    ```
    分析这份销售数据,生成趋势图,
    告诉我关键发现
    ```
  </div>
</Columns>

<Tip>
  HappyCapy 会自动选择最佳工具和方法,你只需要说明目标。
</Tip>

### 2. 提供足够的上下文

<Columns cols={2}>
  <div>
    **不好的提示:**
    ```
    帮我修这个 bug
    ```
  </div>
  <div>
    **好的提示:**
    ```
    用户点击登录按钮后没有反应。

    环境: React 18 + TypeScript
    文件: src/components/LoginForm.tsx
    浏览器控制台显示:
      'Cannot read property of undefined'

    预期行为: 点击后应该发送 API 请求
    并跳转到首页

    请帮我定位问题并修复。
    ```
  </div>
</Columns>

### 3. 迭代式工作,不要要求一次性完美

<Columns cols={2}>
  <div>
    **不好的方式:**
    ```
    帮我构建一个完整的电商网站,包含
    用户认证、产品列表、购物车、支付
    集成、订单管理、评论系统、推荐
    算法...
    ```
  </div>
  <div>
    **好的方式:**
    ```
    先帮我搭建基本框架:
    - 首页展示产品列表
    - 产品详情页
    - 简单的购物车(无需后端)

    完成后我们再逐步添加功能。
    ```
  </div>
</Columns>

**原则:** 拆解大任务 → 逐步实现 → 每步验证

### 4. 利用多模态能力

你可以提供多种格式的输入:

- **截图**: 直接粘贴 (Cmd/Ctrl + V)
- **文件**: 上传 PDF、Excel、Word、图片
- **网页链接**: 分享 URL 进行分析
- **音频/视频**: 提供媒体文件进行转写和分析

### 5. 保存成功的工作流

当一个提示效果很好时:

1. 记录下来 (复制到笔记)
2. 标注使用场景
3. 下次直接复用或微调

<Tip>
  你可以把成功的工作流自定义为技能,所有用户都可以这样做!我们鼓励你创建自己的技能库,让重复任务一键执行。Pro/Max 用户还可以设置定时自动化。
</Tip>

### 6. 定期检查点,方便回滚

对于长时间任务或实验性工作:

```
在实现这个功能前,先让我确认当前
代码都保存好了。
```

**保存策略:**
- 如果使用 Git: 定期提交
- 如果在 HappyCapy 工作区: 定期下载备份

像玩游戏一样存档,出错时可以轻松回滚。

### 7. 明确你的技能水平

在对话开始时说明:

```
我是设计师,不太懂编程。
请用简单语言解释。
```

或:

```
我是高级开发者,熟悉 React 和 TypeScript。
你可以直接给我代码。
```

HappyCapy 会调整沟通方式和细节程度。

### 8. 不要害怕"重新开始"

如果对话方向跑偏了:

<Warning>
  不要试图纠正 AI 的错误方向。直接开新对话,重新描述任务。通常比修正更快。
</Warning>

像使用"老虎机":
1. 保存状态
2. 尝试 30 分钟
3. 满意就接受,不满意就重来

## 常见误区

<AccordionGroup>
  <Accordion icon="warning" title="误区 1: AI 应该一次性给我完美结果">
    **现实:** AI 是协作伙伴,不是魔法棒。

    **正确心态:**
    - 迭代式工作
    - 提供反馈
    - 逐步改进
  </Accordion>

  <Accordion icon="warning" title="误区 2: 我的提示不够'专业',AI 不会理解">

    **正确方法:**
    - 像和朋友聊天一样描述需求
    - 不需要学"提示工程"
    - HappyCapy 会理解你的意图
  </Accordion>

  <Accordion icon="warning" title="误区 3: 我不懂技术,用不了 AI 编程工具">
    **现实:** HappyCapy 是为非技术人员设计的。

    **证据:**
    - 零编程经验的财务团队执行复杂数据工作流
    - 无开发背景的法务团队一小时内构建出可访问的应用
    - 产品设计师直接实现前端功能
  </Accordion>
</AccordionGroup>

  - AI 应该为所有人服务 - 不仅仅是开发者
