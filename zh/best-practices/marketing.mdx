## 营销人员的建议

### 1. 识别支持 API 的重复任务

许多营销平台都有 API,可以实现自动化:

<CardGroup cols={3}>
  <Card title="广告平台" icon="bullhorn">
    Google Ads、Meta Ads、LinkedIn Ads
  </Card>

  <Card title="设计工具" icon="pen-ruler">
    Figma、Canva
  </Card>

  <Card title="分析工具" icon="chart-mixed">
    Google Analytics、Mixpanel
  </Card>
</CardGroup>

<Tip>
  这些平台是自动化工作流的绝佳候选。
</Tip>

### 2. 拆分为专门的子任务

不要一次要求所有内容。将复杂活动分解为步骤:

<AccordionGroup>
  <Accordion icon="heading" title="任务 1: 生成标题">
    ```
    为我们的夏季活动生成 50 个标题变体
    ```
  </Accordion>

  <Accordion icon="paragraph" title="任务 2: 生成描述">
    ```
    生成与这些标题匹配的描述变体
    ```
  </Accordion>

  <Accordion icon="table" title="任务 3: 组合和格式化">
    ```
    将标题和描述组合成所有有效的
    排列组合,输出为 CSV
    ```
  </Accordion>
</AccordionGroup>

<Note>
  这种方法使调试和优化更容易。
</Note>

### 3. 实施前充分头脑风暴

<Columns cols={2}>
  <div>
    **先规划:**
    - 绘制完整工作流
    - 识别数据源
    - 定义输出格式
    - 考虑边缘情况
  </div>
  <div>
    **然后执行:**
    - 逐步实施
    - 测试每个阶段
    - 根据结果改进
    - 不要要求一次性解决方案
  </div>
</Columns>

### 4. 建立记忆系统跨迭代学习

从你的活动中建立知识库:

```
记录我们上次 A/B 测试的结果:

测试期间: 2025 年 1 月 1-15 日
测试的变体:
- "免费试用"信息: 5.2% CTR, 3.1% 转化率
- "限时优惠"信息: 4.1% CTR, 2.4% 转化率

发现: "免费试用"信息的转化率高出 25%。

对于我们的下一个活动,生成强调
"免费试用"信息的变体,采用类似定位。
```

<Tip>
  让 AI 从你的数据中学习,以改进未来的活动表现。
</Tip>

## 真实案例

### 示例 1: 邮件活动变体

```
为我们的产品发布创建 10 个邮件主题行变体:

产品: HappyCapy Max Plan
发布日期: 3 月 15 日
目标: 现有免费用户

要探索的主题:
- 紧迫性(仅限发布日折扣)
- 好奇心(新功能揭示)
- 社会证明(其他用户喜欢什么)
- 价值主张(ROI 计算)

要求:
- 60 字符以内
- 无垃圾词汇
- 适合 A/B 测试(清晰的变体)
- 包括表情符号建议
```

### 示例 2: 落地页文案

```
为我们的新 AI 视频生成功能编写落地页文案:

目标受众: 内容创作者、营销人员

主要优势:
- 从文本提示生成视频
- 40+ 个 AI 模型可用
- 无需视频编辑技能
- 导出为多种格式

需要的部分:
1. 主标题 + 副标题
2. 3 个功能亮点(每个包含标题 + 2 句描述)
3. 社会证明部分(如何展示推荐)
4. FAQ 部分(5 个常见问题)
5. CTA 变体(3 种不同方法)

语气: 专业但平易近人,专注于创意赋能
```

### 示例 3: 社交媒体内容日历

```
创建 2 周的社交媒体内容日历:

平台: Twitter、LinkedIn、Instagram
产品: HappyCapy
目标: 推动注册、展示用例、建立社区

内容组合:
- 40% 教育性(技巧、教程、最佳实践)
- 30% 用例(客户故事、示例)
- 20% 产品更新(功能、改进)
- 10% 社区(用户内容、互动)

每个帖子提供:
- 平台
- 帖子文案
- 图片建议
- 最佳发布时间
- 相关标签
- 包含的链接

输出为电子表格格式。
```

### 示例 4: 广告性能优化

```
分析这些广告活动并建议优化:

[上传广告性能数据 CSV]

当前情况:
- 预算: $10,000/月
- 目标 ROAS: 3.0
- 当前 ROAS: 2.1

请:
1. 识别表现不佳的广告组
2. 分析哪些受众群表现最佳
3. 建议预算重新分配
4. 基于顶级表现者建议新的广告创意方向
5. 识别要暂停的关键词/定位
6. 根据建议估计潜在的 ROAS 改进
```

## 高级营销工作流

### 多渠道活动创建

```
跨所有渠道发起协调活动:

活动: "人人可用的 AI" 2025 年第二季度
信息: 为非技术用户普及 AI 工具
持续时间: 8 周

创建:
1. Google Ads (搜索 + 展示)
   - 20 个搜索广告变体
   - 10 个响应式展示广告(提供文案 + 规格)

2. 社交媒体
   - LinkedIn: 12 个帖子(思想领导力)
   - Twitter: 20 个帖子(技巧 + 互动)
   - Instagram: 8 个帖子(视觉 + 故事)

3. 邮件营销
   - 发布公告邮件
   - 4 封每周培育邮件
   - 冷线索重新激活邮件

4. 落地页
   - 主区域文案
   - 功能亮点
   - 社会证明部分
   - FAQ 部分

保持一致的信息,同时适应每个平台的最佳实践。
```

### 竞争对手分析

```
分析竞争对手的营销策略:

竞争对手: [列出 3-5 个竞争对手]

研究并编制:
1. 他们信息中使用的主要价值主张
2. 定价定位策略
3. 他们关注的目标受众
4. 内容营销主题
5. 广告创意模式(基于广告库研究)
6. 落地页结构和 CTA

交付成果:
- 竞争分析矩阵
- 差距分析(他们做了我们没做的)
- 差异化机会
- 信息建议

使用网络搜索收集当前信息。
```

### 内容再利用

```
将这篇博客文章再利用为多种格式:

[粘贴博客文章内容]

创建:
1. Twitter 线程(10-12 条推文)
2. LinkedIn 文章(重新格式化为平台)
3. Instagram 轮播(6-8 张幻灯片配关键点)
4. 邮件通讯版本(吸引人的格式)
5. 视频脚本(3 分钟说明)
6. 信息图大纲(视觉数据点)

为每个平台调整语气和格式,同时保持核心信息。
```

### 营销归因分析

```
帮我理解营销归因:

[上传带渠道信息的转化数据]

需要的分析:
1. 按渠道的首次触达归因
2. 按渠道的最后触达归因
3. 多点触达归因模型
4. 渠道辅助分析
5. 转化路径模式
6. 最佳营销组合建议

可视化:
- 按渠道归因(饼图)
- 转化路径(桑基图)
- 按渠道的转化时间
- 按渠道的 ROI

提供预算分配建议。
```

### 系统性测试

创建结构化的 A/B 测试:

```
设置 A/B 测试变体:

对照组: 当前标题"提升你的生产力"
测试变量:
- 标题变体(5 个选项)
- CTA 按钮文本(3 个选项)
- 图片风格(2 个选项)

生成测试矩阵显示所有组合,并根据
营销心理学原则预测哪个可能表现最佳。
```
