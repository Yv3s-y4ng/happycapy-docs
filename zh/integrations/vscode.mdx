## 适用场景

有些开发场景下,你可能需要使用本地 IDE:

- 大型项目开发(数百个文件)
- 使用已配置好的本地开发环境
- 需要特定的付费扩展或企业工具
- 习惯本地 IDE 的快捷键和工作流

好消息:通过 **VSCode Remote Tunnels**,轻松连接到 HappyCapy 云端沙箱!

## 3 步完成连接

<Steps>
  <Step title="在 HappyCapy 启动 Tunnel">
    在 HappyCapy 对话框中输入:

    ```
    请帮我启动 VSCode Remote Tunnel(全局模式) --name HappyCapy,并展示 Github 认证链接和验证码
    ```

    HappyCapy 会自动安装并启动 Tunnel,然后返回:

    <Note>
    **记下两个信息:**
    - 授权页面:`https://github.com/login/device`
    - 授权码:`XXXX-XXXX`(8 位代码)
    </Note>
  </Step>

  <Step title="GitHub 授权">
    在浏览器中完成授权(约 1 分钟):

    1. 打开 [https://github.com/login/device](https://github.com/login/device)
    2. 输入 HappyCapy 提供的授权码
    3. 点击 **Authorize**

    <Tip>
    成功标志:浏览器显示 "Success! You may close this page."
    </Tip>
  </Step>

  <Step title="从本地 VS Code 连接">
    <Accordion title="安装 Remote - Tunnels 扩展">
      1. 打开本地 VS Code
      2. 按 `Ctrl + Shift + X` 打开扩展面板
      3. 搜索 **Remote - Tunnels**(Microsoft 官方)
      4. 点击 **Install**
    </Accordion>

    <Accordion title="连接到 Tunnel">
      1. 按 `F1` 打开命令面板
      2. 输入:`Remote-Tunnels: Connect to Tunnel`
      3. 登录 GitHub(首次需要)
      4. 从列表中选择 `HappyCapy`
      5. 等待连接建立

      <Tip>
      成功标志:左下角显示 `>< Remote [HappyCapy]` 绿色标志
      </Tip>
    </Accordion>
  </Step>
</Steps>

## 开始使用

连接成功后,你可以像操作本地一样使用 HappyCapy 沙箱:

### 打开远程文件夹

<Steps>
  <Step title="点击 'Open Folder...'">
    从 VS Code 文件菜单或欢迎页面
  </Step>

  <Step title="输入文件夹路径">
    常用路径:
    - `/home/node` - 你的主目录
    - `/workspace` - 你的工作区目录
  </Step>

  <Step title="信任工作区">
    在提示时点击 "Yes, I trust the authors"
  </Step>
</Steps>

### 打开终端

按 `Ctrl + \`` (反引号)打开远程终端。

<Tip>
**验证连接:** 运行 `pwd` 和 `whoami` 确认你在云端沙箱中
</Tip>

## 快速参考

### HappyCapy Prompt 速查

| 操作 | Prompt |
|--------|--------|
| 启动 Tunnel | `请帮我启动 VSCode Remote Tunnel(全局模式) --name HappyCapy,并展示 Github 认证链接和验证码` |
| 检查状态 | `VSCode Tunnel 还在运行吗?` |
| 停止服务 | `请停止 VSCode Tunnel 服务` |
| 重新启动 | `请重新启动 VSCode Tunnel` |

### VS Code 快捷键

| 操作 | 快捷键 |
|--------|----------|
| 打开文件夹 | `Ctrl + K`,然后 `Ctrl + O` |
| 打开终端 | `Ctrl + \`` |
| 命令面板 | `F1` 或 `Ctrl + Shift + P` |
| 快速打开 | `Ctrl + P` |
| 断开连接 | 点击左下角 `>< Remote` 标志 |

## 故障排除

<AccordionGroup>
  <Accordion title="找不到我的 Tunnel?">
    1. 在 HappyCapy 中问:"VSCode Tunnel 还在运行吗?"
    2. 如果没有运行,重新输入启动 prompt
    3. 检查本地网络连接
    4. 重新登录 GitHub:
       - 按 `F1` → `Remote-Tunnels: Sign out`
       - 然后重新连接并登录
  </Accordion>

  <Accordion title="连接超时">
    尝试这些步骤:
    1. 检查网络连接
    2. 暂时关闭 VPN(如果使用)
    3. 在 HappyCapy 中重启 Tunnel
    4. 更新 VS Code 到最新版本
  </Accordion>

  <Accordion title="无法保存文件">
    1. 检查磁盘空间:在远程终端运行 `df -h`
    2. 检查文件权限:运行 `ls -la`
    3. 尝试重新连接 Tunnel
  </Accordion>

  <Accordion title="扩展不工作">
    某些扩展需要安装在远程服务器上:
    1. 打开扩展面板(`Ctrl + Shift + X`)
    2. 查找带有 "Install in SSH: HappyCapy" 按钮的扩展
    3. 点击在远程服务器上安装
  </Accordion>
</AccordionGroup>
