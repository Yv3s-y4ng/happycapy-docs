连接 GitHub 后,HappyCapy 可以帮你管理代码仓库、创建和查看 Issue、处理 Pull Request、查看 CI/CD 状态等。最棒的是,你只需要用自然语言告诉 HappyCapy 你想做什么就行了!

## 获取 GitHub Token

Token 就像是一把钥匙,让 HappyCapy 可以代表你操作 GitHub。获取很简单:

<Steps>
  <Step title="打开 GitHub Token 设置">
    访问 [https://github.com/settings/tokens](https://github.com/settings/tokens)

    或者:在 GitHub 页面 → 右上角头像 → Settings → 左侧最下方 Developer Settings → Personal access tokens → Tokens (classic)
  </Step>

  <Step title="生成新 Token">
    1. 点击 **Generate new token** → 选择 **Generate new token (classic)**
    2. 填写 **Note** 字段(备注),随便写,比如 "HappyCapy"
    3. **Expiration**(有效期)建议选 "No expiration"(永不过期)
    4. Select scopes(权限):

    <Tip>
    **小白推荐:** 如果是自己使用,可以把所有权限都勾上,省心!如果想精细控制,至少勾选:
    - `repo`(代码仓库)
    - `workflow`(GitHub Actions)
    - `read:org`(读取组织)
    </Tip>

    5. 点击页面底部 **Generate token** 按钮
    6. 复制生成的 Token(以 `ghp_` 开头)
  </Step>

  <Step title="保存您的 Token">
    <Warning>
    **重要:** Token 只显示一次!请立即复制保存到安全的地方(比如密码管理器)。关闭页面后就再也看不到了。
    </Warning>
  </Step>
</Steps>

## 如何使用

HappyCapy 最大的优势是:你不需要记任何命令,直接用自然语言说就行!

### 创建新仓库

你只需要这样说:

```
帮我创建一个 GitHub 仓库,名字叫 my-project

我的 GitHub 账号:https://github.com/你的用户名
Token:ghp_你的token
```

HappyCapy 会自动帮你创建仓库,设置好基本信息。

### 整理对话并提交到仓库

这是最常用的场景!比如你和 HappyCapy 写了一段代码,想保存到 GitHub:

```
帮我把这次对话整理成文档,并提交到我的 GitHub 仓库

仓库:https://github.com/你的用户名/my-project
Token:ghp_你的token
```

HappyCapy 会:
- 整理对话内容
- 生成合适的文件结构
- 提交代码到 GitHub
- 给你返回仓库链接

### 查看和管理 Issue

查看现有 Issue:

```
帮我看看 my-project 仓库有哪些 Issue

仓库:https://github.com/你的用户名/my-project
Token:ghp_你的token
```

创建新 Issue:

```
帮我在 my-project 仓库创建一个 Issue
标题:修复登录按钮 bug
内容:用户点击登录按钮时没有反应

Token:ghp_你的token
```

### 查看 Pull Request

```
帮我看看 my-project 有哪些待审核的 PR

Token:ghp_你的token
```

### 检查 CI/CD 状态

```
帮我检查一下 my-project 的最新 workflow 运行状态

Token:ghp_你的token
```

## 使用技巧

<AccordionGroup>
  <Accordion title="Token 只需要提供一次">
    在同一个对话中,第一次提供 Token 后,后续操作就不需要重复提供了。HappyCapy 会记住你的 Token。
  </Accordion>

  <Accordion title="可以简写仓库名">
    如果是你自己的仓库,可以直接说仓库名,不用每次都写完整 URL:

    ```
    "帮我看看 my-project 的 Issue"
    ```

    而不是:

    ```
    "帮我看看 https://github.com/username/my-project 的 Issue"
    ```
  </Accordion>

  <Accordion title="自然语言的魔力">
    HappyCapy 理解自然语言,所以你可以:
    - "帮我看看最近的提交记录"
    - "创建一个 README 文件并提交"
    - "检查一下测试有没有通过"
    - "把这个功能的代码提交到新分支"

    不需要记命令,像和朋友聊天一样就行!
  </Accordion>
</AccordionGroup>

## 常见问题

<AccordionGroup>
  <Accordion title="Token 安全吗?">
    Token 只在你的会话中使用,HappyCapy 不会存储或分享给其他人。

    **安全建议:**
    - 定期更换 Token(比如每 3 个月)
    - 不要在公开场合分享 Token
    - 如果怀疑泄露,立即在 GitHub 删除旧 Token 并生成新的
  </Accordion>

  <Accordion title="Token 丢了怎么办?">
    没关系!Token 丢了可以:
    1. 回到 GitHub Token 设置页面
    2. 删除旧的 Token
    3. 重新生成一个新的

    就这么简单!
  </Accordion>

  <Accordion title="HappyCapy 能帮我做什么?">
    基本上所有 GitHub 操作都可以:

    - ✅ 创建/删除仓库
    - ✅ 提交代码、创建分支
    - ✅ 管理 Issue 和 PR
    - ✅ 查看提交历史
    - ✅ 检查 CI/CD 状态
    - ✅ 管理仓库设置

    只要你能想到的,HappyCapy 都能帮你做!
  </Accordion>
</AccordionGroup>
