## 什么是 MCP?

MCP(Model Context Protocol,模型上下文协议)是一个开放的协议标准,让 AI 助手能够安全地连接到各种外部工具、数据源和服务。

简单来说,MCP 就像是给 AI 装上了"插件系统",让它能够:

<CardGroup cols={2}>
  <Card title="邮件集成" icon="envelope">
    读写你的 Gmail 邮件
  </Card>

  <Card title="团队聊天" icon="comments">
    在 Slack 上发送消息
  </Card>

  <Card title="笔记管理" icon="note-sticky">
    访问和编辑 Notion 页面
  </Card>

  <Card title="代码管理" icon="github">
    管理 GitHub 仓库和 Issues
  </Card>

  <Card title="持久化记忆" icon="brain">
    在对话间持久化记忆知识图谱
  </Card>

  <Card title="广泛集成" icon="plug">
    连接 1000+ 其他应用和服务
  </Card>
</CardGroup>

## 为什么需要 MCP?

在 HappyCapy 中安装 MCP 后,你可以:

### 自动化日常工作
让 AI 帮你处理邮件、安排日程、整理文档

### 跨平台协作
在一个对话中完成多个应用的操作(比如从 Gmail 读取附件,处理后上传到 Notion)

### 持久化上下文
通过 Memory MCP,AI 能记住项目信息、人物关系等重要内容,跨会话保持记忆

### 简化开发流程
直接在对话中管理 GitHub 仓库、创建 Issue、查看代码

## 快速安装:使用我们的自动化技能

我们为 HappyCapy 开发了一个一键安装 MCP 的技能,可以自动帮你配置 Memory MCP、Composio MCP 和 GitHub Copilot MCP。

<Steps>
  <Step title="安装技能">
    在 HappyCapy 对话中输入:
    ```
    安装技能: https://github.com/Y1fe1-Yang/happycapy-mcp-manager
    ```
  </Step>

  <Step title="运行技能">
    安装完成后输入:
    ```
    /happycapy-mcp-manager
    ```

    或者直接说:
    ```
    帮我安装 Memory MCP
    ```
  </Step>
</Steps>

<Tip>
技能会引导你完成所有配置步骤,包括收集必要的 API Keys、更新配置文件、重启会话等。对于 Memory MCP,无需任何凭证——一键即可安装!
</Tip>

## Memory MCP:强烈推荐

Memory MCP 是一个官方的知识图谱存储 MCP,能为 AI 提供持久化的记忆能力。**这是我们最推荐的 MCP!**

### 为什么需要 Memory MCP?

<CardGroup cols={3}>
  <Card title="跨会话记忆" icon="arrows-rotate">
    AI 能记住你的项目信息、代码架构、人物关系——下次对话时不用重复介绍
  </Card>

  <Card title="知识图谱" icon="diagram-project">
    以实体和关系的形式存储信息,更结构化、更好搜索
  </Card>

  <Card title="长期上下文" icon="timeline">
    对于长期项目,AI 能持续积累对项目的理解
  </Card>

### Memory MCP 使用示例

安装 Memory MCP 后,您可以使用以下 Prompt 来集成历史对话信息到知识图谱中。根据您的需求选择合适的版本:

<AccordionGroup>
  <Accordion icon="list-check" title="完整版(适合首次使用)">
    详细扫描所有历史 session,提取项目、技术栈、能力画像等信息,适合建立完整的知识图谱。

    ```
    把整个系统我的所有历史对话信息集成到 Memory MCP 知识图谱中。

    具体要求:
    1. 扫描本主机下所有历史 session 的工作空间
    2. 分析每个工作空间的项目内容、技术栈、创建时间
    3. 提取关键信息并提炼成结构化的背景知识
    4. 将以下信息存储到知识图谱:
       - 我完成的所有项目
       - 使用过的技术和工具
       - 项目与技术之间的关系
       - 我的技术能力画像和偏好
       - 时间线和活跃周期
    5. 最后生成一份完整的总结报告
    ```
  </Accordion>

  <Accordion icon="bolt" title="简化版(推荐)">
    快速提取历史对话的核心信息,适合大多数用户的日常使用。

    ```
    把本主机下所有历史 session 的对话信息提炼成背景知识,
    集成到 Memory MCP 知识图谱中。
    ```
  </Accordion>

  <Accordion icon="rabbit-running" title="快速版(最简洁)">
    最简单直接的表达,适合快速使用。

    ```
    把之前所有 session 的对话提炼成记忆存到知识图谱
    ```
  </Accordion>
</AccordionGroup>

<Note>
复制上述任意一个代码块中的 Prompt,直接粘贴到对话框即可开始使用。HappyCapy 会自动扫描历史 sessions 并将信息整合到 Memory MCP 知识图谱中。

**重要**: 安装 Memory 之后的对话将自动保存,无需重复此命令。该服务于沙箱内自行完成——HappyCapy 不会使用任何您的信息。
</Note>

## 与 GitHub 集成

连接 GitHub 后,HappyCapy 可以帮你管理代码仓库、创建和查看 Issues、处理 Pull Requests、查看 CI/CD 状态等。

### 获取 GitHub Token

Token 就像一把钥匙,允许 HappyCapy 代表你操作 GitHub。

<Steps>
  <Step title="打开 GitHub Token 设置">
    访问: [https://github.com/settings/tokens](https://github.com/settings/tokens)

    或者: GitHub 页面 → 右上角头像 → Settings → 左下 Developer Settings → Personal access tokens → Tokens (classic)
  </Step>

  <Step title="生成新 Token">
    - 点击 **"Generate new token"** → 选择 **"Generate new token (classic)"**
    - 填写 **Note**(备注),如 "HappyCapy"
    - Expiration 选择 **"No expiration"**(推荐)
    - 选择 scopes/权限
  </Step>

  <Step title="复制并保存 Token">
    点击底部 **"Generate token"** 生成一个以 `ghp_` 开头的字符串
  </Step>
</Steps>

<Warning>
**重要**: Token 只显示一次!请立即复制并保存到安全的地方(如密码管理器)。一旦关闭页面,你再也看不到它。
</Warning>

### 推荐权限

<Tip>
**对于新手**: 如果你是自己使用,可以勾选所有权限,省心!

**对于精细控制**,至少勾选:
- `repo`(代码仓库)
- `workflow`(GitHub Actions)
- `read:org`(读取组织)
</Tip>

### 使用示例

<AccordionGroup>
  <Accordion icon="folder-plus" title="创建新仓库">
    ```
    帮我创建一个 GitHub 仓库叫 my-project

    我的 GitHub 账号: https://github.com/your-username
    Token: ghp_your_token
    ```
  </Accordion>

  <Accordion icon="upload" title="整理并提交到仓库">
    ```
    帮我把这次对话整理成文档,提交到我的 GitHub 仓库

    仓库: https://github.com/your-username/my-project
    Token: ghp_your_token
    ```

    HappyCapy 会:
    - 整理对话
    - 生成合适的文件结构
    - 提交代码到 GitHub
    - 返回仓库链接
  </Accordion>

  <Accordion icon="list" title="查看和管理 Issues">
    查看 Issues:
    ```
    显示 my-project 仓库的 Issues

    仓库: https://github.com/your-username/my-project
    Token: ghp_your_token
    ```

    创建新 Issue:
    ```
    在 my-project 仓库创建一个 Issue
    标题: 修复登录按钮 bug
    描述: 点击登录按钮无响应

    Token: ghp_your_token
    ```
  </Accordion>

  <Accordion icon="code-pull-request" title="查看 Pull Requests">
    ```
    显示 my-project 待处理的 PRs

    Token: ghp_your_token
    ```
  </Accordion>
</AccordionGroup>

<Tip>
**Token 只需提供一次**: 在同一对话中,第一次提供 Token 后,后续操作无需再次提供。HappyCapy 会记住你的 Token。
</Tip>

## 与 Notion 集成

连接 Notion 后,HappyCapy 可以帮你读取、编辑、创建 Notion 页面,自动整理笔记,生成文档等。

### 获取 Notion Token

<Steps>
  <Step title="创建 Notion Integration">
    访问: [https://www.notion.so/my-integrations](https://www.notion.so/my-integrations)

    点击 **"+ New integration"** 按钮
  </Step>

  <Step title="填写 Integration 信息">
    - **Name**: 随便写,如 "HappyCapy"
    - **Associated workspace**: 选择你的工作区
    - **Capabilities**: 默认即可(Read content, Update content, Insert content)
  </Step>

  <Step title="复制 Internal Integration Token">
    创建后,复制 **"Internal Integration Token"**(格式: `ntn_xxxxxxxxxxxx`)
  </Step>
</Steps>

<Warning>
**关键步骤**: 获取 Token 后,你需要授权 Integration!继续下面的步骤。
</Warning>

### 给 Integration 授权访问页面

这是最关键的一步!

<Steps>
  <Step title="打开你的 Notion 页面">
    打开你想让 HappyCapy 访问的页面
  </Step>

  <Step title="添加连接">
    点击右上角 **"···"** 按钮 → 选择 **"Connections"** → 找到你的 Integration(如 "HappyCapy")
  </Step>

  <Step title="确认授权">
    点击确认授权
  </Step>
</Steps>

<Note>
每个你想让 HappyCapy 访问的页面都需要单独授权。如果页面有子页面,子页面会继承权限。
</Note>

### 获取页面 ID

要操作 Notion 页面,你需要知道页面的 ID:

1. 打开你的 Notion 页面
2. 查看浏览器地址栏的 URL
3. URL 最后一段(32 个字符)就是页面 ID

示例 URL:
```
https://www.notion.so/My-Page-3065ab5584b5803e8572fcb253106e5a
```

页面 ID:
```
3065ab5584b5803e8572fcb253106e5a
```

### 在 HappyCapy 中使用

```
读取我的 Notion 页面

Token: ntn_your_token
页面链接: https://www.notion.so/your-page-link
```

或者:

```
帮我更新 Notion 页面的内容

Token: ntn_your_token
页面 ID: 3065ab5584b5803e8572fcb253106e5a
新内容: [你想添加的内容]
```
