## 什么是 Automations?

Automations 是一套自动化规则系统,由 **触发器** 和 **动作** 组成:

<Columns cols={2}>
  <div>
    **触发器**

    什么情况下启动自动化:
    - 收到邮件
    - 文件上传
    - 定时任务
    - Webhook 事件
  </div>

  <div>
    **动作**

    触发后自动执行什么操作:
    - 生成报告
    - 发送通知
    - 处理数据
    - 更新数据库
  </div>
</Columns>

## 如何创建 Automation

<Steps>
  <Step title="打开 Automations">
    点击侧边栏的 **"Automations"**
  </Step>

  <Step title="创建新 Automation">
    点击 **"Create Automation"** 按钮
  </Step>

  <Step title="选择触发条件">
    选择什么事件会启动自动化
  </Step>

  <Step title="定义执行动作">
    指定触发后应该发生什么
  </Step>

  <Step title="保存并激活">
    保存并激活你的自动化规则
  </Step>
</Steps>

## 详细案例:每日新闻精选

<Note>
**设置时间**: 2 分钟 | **后续**: 每天自动运行

**痛点**: 每天打开 10 个网站看新闻,信息太多看不过来,有价值的内容被淹没。
</Note>

### Automation 配置

```
帮我设置一个自动化任务:

名称:每日新闻精选

触发条件:每天早上 8:00

执行流程:
1. 抓取以下领域的热点新闻:
   - AI 和科技
   - 创业和商业
   - 产品和设计

2. 智能过滤:
   - 排除娱乐八卦
   - 排除标题党和广告
   - 只保留有深度的内容

3. AI 总结每条新闻(50 字核心内容)

4. 按重要性排序(优先行业重大事件)

5. 生成精美的 HTML 日报

6. 发送到我的邮箱

让我每天上班路上就能快速了解行业动态!
```

<Tip>
设置完成后,HappyCapy 会每天早上 8 点自动执行。你只需要在手机上打开邮件,5 分钟就能看完一天的重要资讯!
</Tip>

## 触发器类型

Automations 支持多种触发器类型:

<CardGroup cols={2}>
  <Card title="定时" icon="calendar">
    在特定时间或间隔运行(每日、每周、每月)
  </Card>

  <Card title="文件事件" icon="file">
    文件上传或修改时触发
  </Card>

  <Card title="邮件" icon="envelope">
    收到邮件时触发(特定发件人、主题等)
  </Card>

  <Card title="Webhooks" icon="webhook">
    通过 HTTP 请求由外部服务触发

  <Card title="手动" icon="hand-pointer">
    点击按钮按需运行
  </Card>

  <Card title="链式" icon="link">
    另一个 automation 完成时触发
  </Card>
</CardGroup>

## 成功秘诀

<Tip>
**自然语言有效**: 你不需要了解技术细节。只需用简单的语言描述你想要发生什么,HappyCapy 会为你配置 automation。
</Tip>

<Tip>
**迭代改进**: 从基本的 automation 开始,根据结果逐步完善。你随时可以编辑和增强现有的 automations。
</Tip>

<Warning>
**注意频率**: 运行太频繁的 Automations(如每分钟)会消耗大量资源。为定时 automations 选择合适的间隔。
</Warning>

## 故障排除

<AccordionGroup>
  <Accordion icon="circle-question" title="Automation 未触发">
    - 检查 automation 是否已激活(开关打开)
    - 验证触发条件是否正确
    - 查看 automation 日志寻找错误
    - 检查是否授予了所需权限
  </Accordion>

  <Accordion icon="circle-xmark" title="动作执行失败">
    - 检查 automation 日志查看错误消息
    - 验证 API keys 和凭证是否有效
    - 确保所需文件或数据可用
    - 手动测试动作以隔离问题
  </Accordion>

  <Accordion icon="clock-rotate-left" title="Automation 运行太慢">
    - 简化复杂的处理步骤
    - 考虑拆分为多个 automations
    - 减少处理的数据量
    - 使用更高效的文件格式
  </Accordion>
</AccordionGroup>
