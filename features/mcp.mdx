## What Is MCP?

MCP (Model Context Protocol) is an open protocol standard that lets AI assistants securely connect to external tools, data sources, and services.

Think of MCP as a "plugin system" for AI. It lets AI:

<CardGroup cols={2}>
  <Card title="Email Integration" icon="envelope">
    Read and write your Gmail messages
  </Card>

  <Card title="Team Chat" icon="comments">
    Send messages on Slack
  </Card>

  <Card title="Note Management" icon="note-sticky">
    Visit and edit Notion pages
  </Card>

  <Card title="Code Management" icon="github">
    Manage GitHub repositories and Issues
  </Card>

  <Card title="Persistent Memory" icon="brain">
    Knowledge graph memory between conversations
  </Card>

  <Card title="Wide Integration" icon="plug">
    Connect to 1000+ other apps and services
  </Card>
</CardGroup>

## Why Use MCP?

After installing MCP in Happycapy, you can:

### Automate Routine Tasks
Let AI handle emails, schedules, and document organization

### Cross-Platform Collaboration
Do multiple application operations in one conversation (like reading attachments from Gmail, processing them, and uploading to Notion)

### Persistent Context
Memory MCP lets AI remember project information, relationships, and other important content across sessions

### Simplify Development
Manage GitHub repositories, create Issues, and view code directly in conversations

### Memory MCP Usage Examples

After installing Memory MCP, you can use these prompts to integrate historical conversation information into the knowledge graph. Choose the version that fits your needs:

<AccordionGroup>
  <Accordion icon="list-check" title="Full version (for first-time use)">
    Scan all historical sessions in detail and extract information like projects, technology stacks, and capability portraits. Good for building a complete knowledge graph.

    ```
    Integrate all my historical conversation data across the system into the Memory MCP knowledge graph.

    Detailed requirements:
    1. Scan all historical session workspaces on this machine
    2. Analyze each workspace's project content, tech stack, and creation time
    3. Extract key information and structure it into background knowledge
    4. Store the following in the knowledge graph:
       - All projects I have completed
       - Technologies and tools I have used
    ```

  </Accordion>

  <Accordion icon="bolt" title="Simplified version (recommended)">

  <Accordion icon="rabbit-running" title="Quick version (the simplest)">

</AccordionGroup>

</Note>

## Integration with GitHub

After connecting to GitHub, Happycapy can manage code repositories, create and view Issues, process Pull Requests, view CI/CD status, and more.

### Get GitHub Token

A token is like a key that lets Happycapy operate GitHub on your behalf.

<Steps>
  <Step title="Open GitHub token settings">
    Visit: [https://github.com/settings/tokens](https://github.com/settings/tokens)

    Or: GitHub page → avatar in upper right corner → Settings → bottom left Developer Settings → Personal access tokens → Tokens (classic)
  </Step>

  <Step title="Generate new token">
    - Click **"Generate new token"** → select **"Generate new token (classic)"**
    - Fill in **Note** (remarks), such as "Happycapy"
    - Select **"No expiration"** for Expiration (recommended)
    - Select scopes/permissions
  </Step>

  <Step title="Copy and save token">
    Click **"Generate token"** at the bottom to generate a string starting with `ghp_`
  </Step>
</Steps>

<Warning>
**Important**: The token is only displayed once. Copy it immediately and save it in a safe place (like a password manager). Once you close the page, it's gone.
</Warning>

### Recommended Permissions

<Tip>
**For beginners**: If you're using it yourself, you can check all the permissions so you don't have to worry!

**For fine control**, at least check:
- `repo` (code repository)
- `workflow` (GitHub Actions)
- `read:org` (read organization)
</Tip>

### Usage Examples

<AccordionGroup>
  <Accordion icon="folder-plus" title="Create a new repository">
    ```
    Help me create a GitHub repository called my-project

    My GitHub account: https://github.com/your-username
    Token: ghp_your_token
    ```
  </Accordion>

  <Accordion icon="upload" title="Organize and submit to repository">
    ```
    Help me organize this conversation into documentation and commit it to my GitHub repository

    Repository: https://github.com/your-username/my-project
    Token: ghp_your_token
    ```

    Happycapy will:
    - Organize conversations
    - Generate appropriate file structure
    - Submit code to GitHub
    - Return the repository link
  </Accordion>

  <Accordion icon="list" title="View and manage Issues">
    View Issues:
    ```
    Show me the Issues in the my-project repository

    Repository: https://github.com/your-username/my-project
    Token: ghp_your_token
    ```

    Create a new Issue:
    ```
    Create an Issue in the my-project repository
    Title: Fix login button bug
    Description: The login button is unresponsive when clicked

    Token: ghp_your_token
    ```
  </Accordion>

  <Accordion icon="code-pull-request" title="View pull requests">
    ```
    Show me pending PRs for my-project

    Token: ghp_your_token
    ```
  </Accordion>
</AccordionGroup>

<Tip>
**Provide token once**: In the same conversation, after you provide the token the first time, you don't need to provide it again. Happycapy will remember it.
</Tip>

## Integration with Notion

After connecting to Notion, Happycapy can read, edit, create Notion pages, organize notes, generate documents, and more.

### Get Notion Token

<Steps>
  <Step title="Create Notion integration">
    Visit: [https://www.notion.so/my-integrations](https://www.notion.so/my-integrations)

    Click **"+ New integration"** button
  </Step>

  <Step title="Fill in integration information">
    - **Name**: Just write whatever you want, such as "Happycapy"
    - **Associated workspace**: Choose your workspace
    - **Capabilities**: The default is enough (Read content, Update content, Insert content)
  </Step>

  <Step title="Copy internal integration token">
    Once created, copy the **"Internal Integration Token"** (looks like: `ntn_xxxxxxxxxxxx`)
  </Step>
</Steps>

<Warning>
**Key step**: After getting the token, you need to authorize the integration. Continue with the steps below.
</Warning>

### Give Integration Permission to Access Pages

This is the most important step.

<Steps>
  <Step title="Open your Notion page">
    Open the page you want Happycapy to access
  </Step>

  <Step title="Add connection">
    Click the **"···"** button in the upper right corner → Select **"Connections"** → Find your Integration (e.g., "Happycapy")
  </Step>

  <Step title="Confirm authorization">
    Click to confirm authorization
  </Step>
</Steps>

<Note>
Each page you want Happycapy to access requires separate authorization. If the page has subpages, the subpages will inherit permissions.
</Note>

### Get Page ID

To operate on a Notion page, you need to know the page's ID:

1. Open your Notion page
2. View the URL in the browser address bar
3. The last segment (32 characters) of the URL is the page ID

Example URL:
```
https://www.notion.so/My-Page-3065ab5584b5803e8572fcb253106e5a
```

Page ID:
```
3065ab5584b5803e8572fcb253106e5a
```

### Usage in Happycapy

```
Read my Notion page

Token: ntn_your_token
Page link: https://www.notion.so/your-page-link
```

Or:

```
Help me update the content of a Notion page

Token: ntn_your_token
Page ID: 3065ab5584b5803e8572fcb253106e5a
New content: [content you want to add]
```
