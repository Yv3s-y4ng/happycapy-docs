## 高效使用的黃金法則

### 1. 清楚描述你的目標,而非過程

<Columns cols={2}>
  <div>
    **不好的提示:**
    ```
    先用 pandas 讀取 CSV,
    然後用 matplotlib 畫圖
    ```
  </div>
  <div>
    **好的提示:**
    ```
    分析這份銷售資料,生成
    趨勢圖,告訴我關鍵發現
    ```
  </div>
</Columns>

<Tip>
  HappyCapy 會自動選擇最佳工具和方法。你只需要說明目標。
</Tip>

### 2. 提供足夠的上下文

<Columns cols={2}>
  <div>
    **不好的提示:**
    ```
    請幫我修復這個 bug
    ```
  </div>
  <div>
    **好的提示:**
    ```
    使用者點擊登入按鈕後沒有反應。

    環境:React 18 + TypeScript
    檔案:src/components/LoginForm.tsx
    瀏覽器主控台顯示:
      'Cannot read property of undefined'

    預期行為:點擊後應該發送 API 請求並
    跳轉到首頁

    請幫我定位問題並修復。
    ```
  </div>
</Columns>

### 3. 迭代式工作,不要要求一次性完美

<Columns cols={2}>
  <div>
    **不好的做法:**
    ```
    請幫我搭建一個完整的電商網站,包含
    使用者認證、產品列表、購物車、支付
    整合、訂單管理、評論系統、推薦
    算法...
    ```
  </div>
  <div>
    **好的做法:**
    ```
    先請幫我搭建基本框架:
    - 首頁展示產品列表
    - 產品詳情頁
    - 簡單的購物車(無需後端)

    完成後我們再逐步添加功能。
    ```
  </div>
</Columns>

**原則:** 拆解大任務 → 逐步實現 → 每步驗證

### 4. 利用多模態能力

你可以提供多種格式的輸入:

- **截圖**: 直接貼上(Cmd/Ctrl + V)
- **檔案**: 上傳 PDF、Excel、Word、圖片
- **網頁連結**: 分享 URL 進行分析
- **音訊/影片**: 提供媒體進行轉寫和分析

### 5. 保存成功的工作流

當一個提示效果很好時:

1. 記錄下來(複製到筆記)
2. 標記使用場景
3. 下次直接複用或微調

<Tip>
  你可以把成功的工作流自訂為 Skills,所有使用者都可以這樣做!我們鼓勵你建立自己的技能庫,讓重複任務一鍵執行。Pro/Max 使用者還可以設定定時自動化。
</Tip>

### 6. 定期檢查點,方便回滾

對於長時間任務或實驗性工作:

```
在實現這個功能前,先讓我確認當前程式碼都保存好了。
```

**保存策略:**
- 如果使用 Git: 定期 commit
- 如果在 HappyCapy 工作區: 定期下載備份

像玩遊戲一樣存檔,出錯時可以輕鬆回滾。

### 7. 明確你的技術水平

在對話開始時說明:

```
我是設計師,不太懂程式,請用簡單語言解釋。
```

或:

```
我是高級開發者,熟悉 React 和 TypeScript。
你可以直接給我程式碼。
```

HappyCapy 會調整溝通方式和細緻程度。

### 8. 不要害怕「重新開始」

如果對話方向跑偏了:

<Warning>
  不要試圖糾正 AI 的錯誤。直接開新對話,重新描述任務。通常比糾正更快。
</Warning>

像使用「老虎機」:
1. 保存狀態
2. 嘗試 30 分鐘
3. 滿意就接受;不滿意就重來

## 常見誤區

<AccordionGroup>
  <Accordion icon="warning" title="誤區 1: AI 應該一次性給我完美結果">
    **現實:** AI 是協作夥伴,不是魔法棒。

    **正確心態:**
    - 迭代式工作
    - 提供回饋
    - 逐步改進
  </Accordion>

  <Accordion icon="warning" title="誤區 2: 我的提示不夠「專業」,AI 不會理解">
    **現實:** 自然語言就是最好的提示。

    **正確方法:**
    - 像和朋友聊天一樣描述需求
    - 不需要學習「提示工程」
    - HappyCapy 會理解你的意圖
  </Accordion>

  <Accordion icon="warning" title="誤區 3: 我不懂技術,不能用 AI 程式工具">
    **現實:** HappyCapy 專為非技術人員設計。

    **證據:**
    - 財務團隊零程式經驗,執行複雜資料工作流
    - 法律團隊無開發背景,一小時搭建無障礙應用
    - 產品設計師直接實現前端功能
  </Accordion>
</AccordionGroup>
