## Desktop 是什麼

Desktop 是一個有名稱的專案工作區。每個 Desktop 在沙箱裡有一個專屬目錄，路徑為 `~/a0/workspace/<desktop-id>/`。在這個 Desktop 裡開啟的所有工作階段都在這個共享目錄裡工作——它們讀寫的是同一批檔案。

這讓 Desktop 非常適合跨多次對話的專案。在一個工作階段裡開始的任務，換一個工作階段接著做，檔案還在原地。

## 工作階段（Sessions）

每個 Desktop 裡可以同時執行多個工作階段。每個工作階段是獨立的對話串，但它們都共享同一個專案目錄。

實用場景：
- 一個工作階段生成圖片，另一個同時寫配套文案
- 一個工作階段處理前端程式碼，另一個處理後端
- 一個工作階段做研究，另一個起草報告

所有輸出都落在同一個 `~/a0/workspace/<desktop-id>/` 目錄裡。

**新建工作階段：** 點擊 Desktop 頂部工作階段名稱旁邊的 **+** 按鈕。

**切換工作階段：** 點擊頂部的工作階段名稱下拉選單選擇。

## 資料夾（Folders）

資料夾用來在側邊欄裡給 Desktop 分組。點擊「Desktops」旁邊的 **+** 圖示（Create Folder）新建。

範例結構：

```
📁 開發
  ├── 我的應用 前端
  ├── 我的應用 後端
  └── 部署腳本

📁 內容創作
  ├── 部落格文章
  └── 社群媒體
```

資料夾只用於展示分組，不影響檔案系統。

## 收藏（Favorites）

點擊工作階段標題列裡的 **☆** 圖示，把常用工作階段固定到側邊欄頂部的 **Favorites** 區域。再點一次取消收藏。

## 新建 Desktop

點擊側邊欄裡的 **New Desktop**，起一個能反映專案內容的名稱。Happycapy 會為它建立一個全新的工作區目錄。

<Tip>
  一個專案對應一個 Desktop 是個好習慣。需要並行推進任務時，在同一個 Desktop 下開多個工作階段——所有輸出都落在同一個目錄裡，結果不會散落各處。
</Tip>
