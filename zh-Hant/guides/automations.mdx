## 什麼是 Automations

Automations 是一套自動化規則系統,由**觸發器**和**動作**組成:

<Columns cols={2}>
  <div>
    **觸發器**

    什麼情況下啟動自動化:
    - 收到郵件
    - 檔案上傳
    - 定時任務
    - Webhook 事件
  </div>

  <div>
    **動作**

    觸發後自動執行什麼操作:
    - 生成報告
    - 發送通知
    - 處理資料
    - 更新資料庫
  </div>
</Columns>

## 如何建立 Automation

<Steps>
  <Step title="開啟 Automations">
    點選側邊欄的 **「Automations」**
  </Step>

  <Step title="建立新 Automation">
    點選 **「Create Automation」** 按鈕
  </Step>

  <Step title="選擇觸發條件">
    選擇什麼事件將啟動自動化
  </Step>

  <Step title="定義執行動作">
    指定觸發後應該發生什麼
  </Step>

  <Step title="保存並啟用">
    保存並啟用你的自動化規則
  </Step>
</Steps>

## 詳細範例:每日新聞精選

<Note>
**設置時間**:2 分鐘 | **後續**:每天自動執行

**痛點**:每天打開 10 個網站看新聞,資訊太多看不過來,有價值的內容被淹沒
</Note>

### Automation 設定

```
請幫我設置一個自動化任務:

名稱:每日新聞精選

觸發條件:每天早上 8:00

執行流程:
1. 抓取以下領域的熱點新聞:
   - AI 和科技
   - 創業和商業
   - 產品和設計

2. 智能過濾:
   - 排除娛樂八卦
   - 排除標題黨和廣告
   - 只保留有深度的內容

3. AI 總結每條新聞(50 字核心內容)

4. 按重要性排序(優先行業重大事件)

5. 生成精美的 HTML 日報

6. 發送到我的郵箱

讓我每天上班路上就能快速瞭解行業動態!
```

<Tip>
設置完成後,HappyCapy 會每天早上 8 點自動執行。你只需要在手機上打開郵件,5 分鐘就能看完一天的重要資訊!
</Tip>

## 觸發器類型

Automations 支援多種觸發器類型:

<CardGroup cols={2}>
  <Card title="排程" icon="calendar">
    在特定時間或間隔執行(每日、每週、每月)
  </Card>

  <Card title="檔案事件" icon="file">
    當檔案上傳或修改時觸發
  </Card>

  <Card title="郵件" icon="envelope">
    收到郵件時觸發(特定寄件者、主旨等)
  </Card>

  <Card title="Webhooks" icon="webhook">
    通過 HTTP 請求由外部服務觸發
  </Card>

  <Card title="手動" icon="hand-pointer">
    按一下按鈕即可隨需執行
  </Card>

  <Card title="鏈式" icon="link">
    當另一個 automation 完成時觸發
  </Card>
</CardGroup>

## 成功技巧

<Tip>
**自然語言有效**:你不需要了解技術細節。只需用簡單的語言描述你想要發生的事情,HappyCapy 就會為你配置 automation。
</Tip>

<Tip>
**迭代改進**:從基本的 automation 開始,根據結果隨時間優化。你可以隨時編輯和增強現有的 automation。
</Tip>

<Warning>
**注意頻率**:執行頻率過高的 automation(例如每分鐘一次)可能會消耗大量資源。為排程的 automation 選擇適當的間隔。
</Warning>

## 疑難排解

<AccordionGroup>
  <Accordion icon="circle-question" title="Automation 沒有觸發">
    - 檢查 automation 是否已啟用(開關是否開啟)
    - 驗證觸發條件是否正確
    - 檢視 automation 日誌查看錯誤
    - 檢查是否已授予必要的權限
  </Accordion>

  <Accordion icon="circle-xmark" title="動作執行失敗">
    - 檢查 automation 日誌查看錯誤訊息
    - 驗證 API 金鑰和憑證是否有效
    - 確保必要的檔案或資料可用
    - 手動測試動作以隔離問題
  </Accordion>

  <Accordion icon="clock-rotate-left" title="Automation 執行過慢">
    - 簡化複雜的處理步驟
    - 考慮拆分為多個 automation
    - 減少正在處理的資料量
    - 使用更有效率的檔案格式
  </Accordion>
</AccordionGroup>
