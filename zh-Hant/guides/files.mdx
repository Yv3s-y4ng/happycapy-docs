## Files 是什麼

Files 是沙箱的完整檔案瀏覽器。點擊左側邊欄的 **Files** 即可開啟。

你的沙箱是一個持久化的 Linux 環境——Happycapy 建立的每個檔案、執行的每個腳本、產生的每個輸出都保存在這裡。Files 讓你直接存取這個檔案系統。

## 左側導覽

左側面板分三個區塊：

**Home** — 沙箱檔案系統的根目錄。設定檔、已安裝的工具、對話輸出、Happycapy 在你各次對話中建立的檔案，都在這裡。

**Desktops** — 按桌面整理的檔案視圖。如果你有多個桌面對應不同專案，這個視圖能幫你快速看到哪些檔案屬於哪個桌面。

**Skills** — 沙箱中已安裝的 Skills。可以在這裡瀏覽 Skill 檔案，查看某個 Skill 的內容或複製它的設定。

## 尋找檔案

右上角的 **搜尋框** 支援按檔案名稱搜尋。檔案列表會顯示名稱、修改日期和檔案大小。

搜尋框旁邊的圖示可以切換格狀視圖和列表視圖。

## 找到輸出檔案

Happycapy 完成任務後產生的檔案——圖片、文件、資料匯出、處理過的影片——都會保存在沙箱裡。在 Files 裡找到它們，右鍵下載到本機，或者直接讓 Happycapy 打包：

```
把 outputs 資料夾裡的圖片打包成 zip 檔案，讓我可以下載
```

## 上傳檔案

點擊對話輸入框旁的 **+** 按鈕，從本機選擇檔案上傳到對話。上傳的檔案會放在沙箱 Home 目錄的 `uploads/` 資料夾中。

<Note>
  支援的格式：圖片（JPG、PNG）、文件（PDF、Word、Excel）、影片（MP4）、資料檔案（CSV、JSON）、程式碼（所有語言）。單個檔案最大 100MB。
</Note>

## 進階：設定你的專案工作區

每個 Desktop 在沙箱裡有一個專屬的專案目錄，路徑為 `~/a0/workspace/<desktop-id>/`。同一 Desktop 下的所有工作階段共享這個目錄。目錄裡的所有檔案都可以直接編輯——在 Files 裡導覽到這裡，修改任何東西都行。

這讓你對 Claude 在這個 Desktop 裡的行為有完整的控制權。一個結構清晰的專案工作區長這樣：

```
你的專案/
├── CLAUDE.md              ← 對這個 Desktop 裡 Claude 的持久指令
├── README.md
│
├── .claude/
│   ├── settings.json      ← Claude Code 設定
│   ├── hooks/             ← 自動化觸發器和檢查
│   └── skills/            ← 這個專案專用的 Skills
│       └── my-skill/
│           └── SKILL.md
│
├── docs/
│   ├── architecture.md
│   └── decisions/
│
└── src/                   ← 實際的專案程式碼
    └── api/
        └── CLAUDE.md      ← 這個模組的子脈絡
```

**`CLAUDE.md`** 是最有用的檔案。把專案背景、程式碼規範和持久指令寫進去。Claude 在這個 Desktop 的每次工作階段開始時都會讀取它，你就不用每次重新介紹專案了。

**`.claude/skills/`** 存放這個專案專用的 Skills，只對這個 Desktop 的工作階段生效，不會全域安裝。

**`.claude/hooks/`** 存放自動化腳本，在特定觸發條件下執行——例如每次編輯程式碼後自動跑測試。

對話歷史單獨儲存在 `~/.claude/projects/` 下，按 Desktop 分資料夾，格式為 `.jsonl` 檔案。
