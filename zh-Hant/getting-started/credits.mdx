## 在哪裡查看用量

點擊左側邊欄的 **Usage**。頂部會顯示兩個數字：選定時間段內消耗的積分總量和請求總次數。

右上角的 **7 Days / 30 Days / 90 Days** 按鈕用來切換時間範圍。

## 讀懂圖表

**Credits by Model** 是按天顯示的積分消耗堆疊長條圖，按模型分色。當天的柱子越高說明活動越多，圖例會顯示哪些模型帶來了這些消耗——Claude 系列、圖片生成模型、影片生成模型各有對應顏色。

**Requests by Model** 結構相同，但縱軸是請求次數而非積分成本。某個模型可能請求次數多但積分消耗少（比如 Haiku），也可能反過來（比如 Opus）。

## 積分怎麼計算

不同模型和任務類型的積分消耗速率不同：

- **Opus 4.6** 每次請求消耗最多——能力最強，適合複雜推理任務
- **Sonnet 4.6** 是預設模型，能力與成本之間取得平衡
- **Haiku 4.5** 最便宜，適合快速簡單的回答
- **MiniMax M2.5 / M2.7** 日常任務的高性價比選擇
- **圖片和影片生成**（Gemini、Seedream 等）與文字請求分開計算積分，查看圖表可以了解媒體生成佔了多少用量

## 選對模型

每個對話輸入框右下角都有模型選擇器，按需選擇：

| 任務 | 推薦模型 |
|------|---------|
| 複雜分析、程式設計、推理 | Opus 4.6 |
| 大多數日常任務 | Sonnet 4.6 |
| 快速提問、簡單修改 | Haiku 4.5 |
| 大量任務、控制成本 | MiniMax M2.7 |

使用 Skills 時，Happycapy 會自動選擇合適的模型——圖片生成任務走圖片模型，影片任務走影片模型。

## Max 方案

Max 方案用戶的 Token 無限制。Usage 頁面仍會顯示你的消耗量供參考，但沒有上限，不用擔心任務做到一半跑出額度。

<Tip>
  想節省積分，可以用 Sonnet 而非 Opus 來跑複雜的多步驟任務，除非你明確需要 Opus 的推理深度。研究類任務中的輕量步驟交給 Haiku 處理效果也不錯。
</Tip>
