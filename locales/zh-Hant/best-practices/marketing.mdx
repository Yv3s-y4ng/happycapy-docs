提供全面的規格進行批量生成:

```

目標受眾: 中小企業主、自由職業者

核心賣點:
- 零技術門檻
- 提高工作效率 3 倍
- 在雲端運行,手機也能用
- 生成圖片、處理文件、自動化工作流程

- 標題 1: 30 字元或更少

- 20 個突出「節省時間」
```

## 給營銷人員的建議

### 1. 識別有 API 支援的重複任務

許多營銷平台都有 API,可以實現自動化:

<CardGroup cols={3}>
  <Card title="廣告平台" icon="bullhorn">
    Google Ads、Meta Ads、LinkedIn Ads
  </Card>

  <Card title="設計工具" icon="pen-ruler">
    Figma、Canva
  </Card>

  <Card title="分析" icon="chart-mixed">
    Google Analytics、Mixpanel
  </Card>
</CardGroup>

<Tip>
  這些平台是自動化工作流程的絕佳候選者。
</Tip>

### 2. 拆分為專門的子任務

不要一次要求所有內容。將複雜的活動分解為步驟:

<AccordionGroup>
  <Accordion icon="heading" title="任務 1: 生成標題">
    ```
    為我們的夏季活動生成 50 個標題變體
    ```
  </Accordion>

  <Accordion icon="paragraph" title="任務 2: 生成描述">
    ```
    生成與這些標題匹配的描述變體
    ```
  </Accordion>

  <Accordion icon="table" title="任務 3: 組合和格式化">
    ```
    將標題和描述組合成所有有效排列,
    輸出為 CSV
    ```
  </Accordion>
</AccordionGroup>

<Note>
  這種方法使調試和優化更容易。
</Note>

### 3. 實施前徹底腦力激盪

<Columns cols={2}>
  <div>
    **先計劃:**
    - 規劃完整工作流程
    - 識別資料來源
    - 定義輸出格式
    - 考慮邊緣情況
  </div>
  <div>
    **然後執行:**
    - 逐步實施
    - 測試每個階段
    - 根據結果改進
    - 不要要求一次性解決方案
  </div>
</Columns>

### 4. 建立記憶系統以跨迭代學習

從你的活動中建立知識庫:

```
記錄我們上次 A/B 測試的結果:

測試期間: 2025 年 1 月 1-15 日
測試的變體:
- 「免費試用」訊息: 5.2% CTR,3.1% 轉化
- 「限時優惠」訊息: 4.1% CTR,2.4% 轉化

發現:「免費試用」訊息在轉化率上表現好 25%。

對於我們的下一個活動,生成強調
「免費試用」訊息的變體,定位類似。
```

<Tip>
  讓 AI 從你的資料中學習,以改善未來的活動效果。
</Tip>

## 實際範例

### 範例 1: 電子郵件活動變體

```
為我們的產品發布創建 10 個電子郵件主題行變體:

產品: Happycapy Max Plan
發布日期: 3 月 15 日
目標: 現有免費使用者

要探索的主題:
- 緊迫性(僅發布日當天折扣)
- 好奇心(新功能揭示)
- 社會證明(其他使用者喜歡的內容)
- 價值主張(ROI 計算)

要求:
- 60 字元或更少
- 無垃圾詞彙
- A/B 測試友好(明確變體)
- 包括表情符號建議
```

### 範例 2: 落地頁文案

```
為我們的新 AI 影片生成功能編寫落地頁文案:

目標受眾: 內容創作者、營銷人員

主要優勢:
- 從文字提示生成影片
- 40+ AI 模型可用
- 無需影片編輯技能
- 多種格式導出

需要的區塊:
1. 英雄標題 + 副標題
2. 3 個功能亮點(標題 + 2 句描述每個)
3. 社會證明區塊(如何框架推薦)
4. FAQ 區塊(5 個常見問題)
5. CTA 變體(3 種不同方法)

語調: 專業但平易近人,專注於創意賦能
```

### 範例 3: 社交媒體內容日曆

```
創建 2 週的社交媒體內容日曆:

平台: Twitter、LinkedIn、Instagram
產品: Happycapy
目標: 推動註冊、展示用例、建立社群

內容組合:
- 40% 教育性(提示、教程、最佳實踐)
- 30% 用例(客戶故事、範例)
- 20% 產品更新(功能、改進)
- 10% 社群(使用者內容、參與)

為每篇貼文提供:
- 平台
- 貼文文案
- 圖片建議
- 最佳發布時間
- 相關標籤
- 要包含的連結

以電子表格格式輸出。
```

### 範例 4: 廣告效果優化

```
分析這些廣告活動並建議優化:

[上傳廣告效果資料 CSV]

當前情況:
- 預算: $10,000/月
- 目標 ROAS: 3.0
- 當前 ROAS: 2.1

請:
1. 識別表現不佳的廣告組
2. 分析哪些受眾細分表現最佳
3. 建議預算重新分配
4. 根據表現最佳者建議新的廣告創意方向
5. 識別要暫停的關鍵字/定位
6. 估計建議的潛在 ROAS 改進
```

## 進階營銷工作流程

### 多管道活動創建

```
跨所有管道啟動協調活動:

活動:「人人都能用的 AI」2025 年第二季
訊息: 為非技術使用者民主化 AI 工具
持續時間: 8 週

創建:
1. Google Ads(搜尋 + 展示)
   - 20 個搜尋廣告變體
   - 10 個響應式展示廣告(提供文案 + 規格)

2. 社交媒體
   - LinkedIn: 12 篇貼文(思想領導力)
   - Twitter: 20 篇貼文(提示 + 參與)
   - Instagram: 8 篇貼文(視覺 + 故事)

3. 電子郵件營銷
   - 發布公告電子郵件
   - 4 封每週培育電子郵件
   - 冷線索重新參與電子郵件

4. 落地頁
   - 英雄區塊文案
   - 功能亮點
   - 社會證明區塊
   - FAQ 區塊

在適應每個平台的最佳實踐的同時保持一致的訊息。
```

### 競爭對手分析

```
分析競爭對手的營銷策略:

競爭對手: [列出 3-5 個競爭對手]

研究並編譯:
1. 他們訊息中使用的主要價值主張
2. 定價定位策略
3. 他們專注的目標受眾
4. 內容營銷主題
5. 廣告創意模式(基於廣告庫研究)
6. 落地頁結構和 CTA

交付成果:
- 競爭分析矩陣
- 差距分析(他們做的我們沒做的)
- 差異化機會
- 訊息建議

使用網頁搜尋收集當前信息。
```

### 內容再利用

```
將此部落格貼文重新用於多種格式:

[貼上部落格貼文內容]

創建:
1. Twitter 線程(10-12 條推文)
2. LinkedIn 文章(為平台重新格式化)
3. Instagram 輪播(6-8 張幻燈片,關鍵點)
4. 電子郵件通訊版本(吸引人的格式)
5. 影片腳本(3 分鐘解說)
6. 資訊圖表大綱(視覺資料點)

在保持核心訊息的同時為每個平台調整語調和格式。
```

### 營銷歸因分析

```
幫我理解營銷歸因:

[上傳包含管道信息的轉化資料]

需要的分析:
1. 按管道的首次接觸歸因
2. 按管道的最後接觸歸因
3. 多接觸歸因模型
4. 管道輔助分析
5. 轉化路徑模式
6. 最佳營銷組合建議

可視化:
- 按管道歸因(餅圖)
- 轉化路徑(桑基圖)
- 按管道轉化時間
- 按管道 ROI

提供預算分配建議。
```

## 營銷自動化最佳實踐

### 版本控制你的文案

<Tip>
  跟蹤獲勝變體並建立高效能文案的滑動檔案。在未來的提示中參考這些:

  ```
  生成類似於我們表現最佳廣告的新變體:
  [貼上獲勝文案]

  保持相同的語調和結構,但探索新角度。
  ```
</Tip>

### 系統化測試

創建結構化的 A/B 測試:

```
設定 A/B 測試變體:

對照組: 當前標題「提升你的生產力」
測試變數:
- 標題變體(5 個選項)
- CTA 按鈕文字(3 個選項)
- 圖片風格(2 個選項)

生成顯示所有組合的測試矩陣,並根據
營銷心理學原則預測哪些可能表現最佳。
```

### 自動化報告

```
創建自動化每週營銷報告:

資料來源:
- Google Ads 效果(通過 API 或 CSV 導出)
- Facebook Ads 資料
- 網站分析
- 電子郵件活動指標

報告區塊:
1. 執行摘要(關鍵指標 vs. 目標)
2. 管道效果(週環比)
3. 表現最佳內容
4. 預算進度
5. 下週建議

格式化為帶有圖表和表格的 PDF。
每週一上午 9 點自動執行。(Pro/Max 方案)
```
